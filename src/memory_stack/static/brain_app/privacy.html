Privacy

Brain stores personal memory only after authenticated use and explicit write confirmation on the ChatGPT App surface.

Stored data can include memories, profile context, open loops, source metadata, Palate preference records, and auditable ingestion records. Brain scopes these records to the authenticated user and does not expose raw admin tools on the public app surface.

When Brain is used from ChatGPT, ChatGPT receives minimized tool responses needed to complete the requested memory workflow. The public app surface strips internal user ids, session ids, OAuth client ids, request ids, raw metadata JSON, datasets, timestamps, tokens, and password fields from app-facing payloads.

Brain may use configured model, embedding, search, and place-enrichment providers to classify, retrieve, summarize, or enrich records. Do not store passwords, API keys, OAuth tokens, or other secrets in Brain memories.

Records are retained until the authenticated user or operator removes them, or until an environment backup/retention policy expires. User-level review, profile, and undo controls are available in the dashboard; operators can also run authenticated deletion workflows.

Authentication passwords are stored as Argon2id hashes in the Brain user registry. Browser sessions use an HTTP-only cookie and CSRF token; MCP clients use OAuth bearer tokens.

Use the dashboard review, profile, account, and undo controls to inspect, change, export-preview, or remove recent data. For support or deletion help, use the support page or email support@dceb.net.
